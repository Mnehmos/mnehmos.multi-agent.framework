# Code of Conduct

This project is a public, reusable template intended for professional, open collaboration. All participants are expected to contribute in a way that is respectful, inclusive, and aligned with a safe community environment.

This Code of Conduct is intentionally concise and project-agnostic, based on the spirit of the Contributor Covenant.

## Our Pledge

We pledge to make participation in this project a harassment-free experience for everyone, regardless of personal characteristics or background. We welcome contributions that improve clarity, safety, and robustness of the framework.

## Our Standards

Examples of behavior that contributes to a positive environment include:

- Using welcoming and inclusive language.
- Being respectful of differing viewpoints and experiences.
- Providing constructive feedback grounded in technical merit.
- Focusing on improving the framework’s quality, safety, and clarity.

Examples of unacceptable behavior include:

- Harassment, discrimination, or personal attacks.
- Public or private intimidation or threats.
- Disruptive, insulting, or derogatory comments.
- Any behavior that makes collaboration unsafe or hostile.

## Scope

This Code of Conduct applies within:

- GitHub issues, discussions, and pull requests.
- Any project-related communication channels where this framework is discussed or maintained.

Downstream users who adopt this template are encouraged to:

- Keep a clear, public Code of Conduct in their derived repositories.
- Adapt these guidelines as needed while maintaining a safe environment.

## Enforcement

Project maintainers may take appropriate action for any behavior they deem inappropriate, threatening, offensive, or harmful, including:

- Warnings.
- Restricting or removing participation.
- Reporting egregious behavior to relevant platforms or authorities.

For concerns about conduct in this repository:

- Use GitHub’s reporting tools or contact the maintainers privately (for example, via a security/contact email or organization channel defined in the downstream repo).
